SECTION 00

접속과 전체 구조

이 프로그램은 GitHub Pages에 올려 온라인으로 운영하는 웹 도구입니다. 별도 설치 없이 주소 접속만으로 사용할 수 있습니다.

🌐 접속 방식

학생에게 공유할 주소는 GitHub Pages 주소입니다. 수업 전 주소 또는 QR코드를 학급 단체방, LMS, 칠판 화면에 공유하면 됩니다.

운영 기준이전 안내서에 있던 Live Server, python -m http.server, localhost 설명은 삭제했습니다. 학생은 온라인 주소로 바로 접속합니다.

전체 흐름

1
탐구 설계 시작하기

진로 목표, 학생부 기록, 교과 개념을 바탕으로 탐구 계획서를 만듭니다.

2
탐구기록지 작성하기

완성된 설계 내용을 바탕으로 탐구 과정, 결과, 성찰을 기록합니다.

SECTION 01

시작 화면 — 두 개의 큰 선택 카드

처음 접속하면 바로 보이는 화면입니다. 실제 화면은 아래처럼 탐구 설계 시작하기와 탐구기록지 작성하기 두 카드로 구성됩니다.

학생 안내 멘트“먼저 탐구 설계 시작하기를 눌러 계획서를 만들고, 계획서가 끝난 뒤 탐구기록지 작성하기로 넘어가면 된다.”
SECTION 02

SECTION 03

2학년·3학년 탐구 설계 9단계

2학년 탐구 설계와 3학년 심화탐구는 같은 9단계 구조를 사용합니다. 다만 2학년은 탐구의 씨앗 만들기, 3학년은 누적 기록의 심화·전공연계성 강화에 초점이 있습니다.

9단계 핵심 요약

단계화면명해야 할 일
1학생부 기록 정리이전 활동과 관심사를 요약합니다.
2흐름 분석강점 교과, 진로 관심, 탐구 성향을 정리합니다.
3AI 프롬프트AI에게 보낼 질문을 생성하고 복사합니다.
4주제 DB 연계내장 탐구 주제 DB에서 유사 주제를 참고합니다.
5위험도 점검범위 과다, 장비 문제, 개인정보 위험 등을 점검합니다.
6별점 평가후보 주제를 기준별로 비교합니다.
7계획서 작성최종 탐구계획서 12개 항목을 완성합니다.
8세특 포인트수업 중 관찰할 장면을 정리합니다.
9최종 점검필수 항목 확인 후 탐구기록지로 이동합니다.
중요AI 프롬프트는 “생성형 AI에게 맡기기”가 아니라 “탐구 질문을 정교화하기 위한 초안 생성”입니다. 결과를 그대로 복사하지 않도록 안내해야 합니다.
SECTION 04

주제 DB 검색 화면

주제 DB 탭에서는 내장된 탐구 주제를 자료집, 교과군, 난이도, 탐구유형, 기간, 키워드로 검색합니다.

검색 팁

  • 처음에는 키워드 하나만 입력합니다. 예: 환경, 수질, 심박, 센서, 인공지능
  • 결과가 많으면 교과군과 난이도를 함께 조정합니다.
  • “내 기록으로 추천”은 학생이 입력한 관심사와 키워드가 어느 정도 채워진 뒤 사용하는 것이 좋습니다.

교사 모드 5가지 기능

탭활용법
수업 운영안45분·2차시·3차시 운영 흐름을 확인합니다.
위험도 기준주제가 너무 넓거나 장비·개인정보 문제가 있는지 판단합니다.
피드백 생성학생 입력 내용을 바탕으로 피드백 초안을 만듭니다.
세특 관찰수업 중 관찰해야 할 발언·수정·자료 분석 장면을 확인합니다.

교사 모드 주의피드백 초안과 세특 관찰 포인트는 자동 생성 참고자료입니다. 실제 학생 발언과 활동 장면을 교사가 반드시 추가해야 합니다.
SECTION 06

탐구기록지

탐구 설계가 끝난 뒤 과정·결과·성찰을 기록하는 단계입니다. 시작 화면의 STEP 2 또는 설계 화면 STEP 9의 이동 버튼으로 접근합니다.

기록지 작성 흐름

1
설계 불러오기

탐구 설계에서 전달된 제목, 질문, 교과 개념을 불러옵니다.

2
탐구 과정 기록

자료 수집, 분석, 시행착오, 피드백 반영을 순서대로 적습니다.

3
결과와 성찰 작성

예상과 다른 결과, 배운 점, 다음 탐구 방향을 기록합니다.

4
저장·제출

자동 저장과 별도로 JSON 또는 지정 방식으로 제출합니다.

팝업 차단 안내“실제 기록지 열기”를 눌렀는데 아무 창도 열리지 않으면 브라우저 팝업 차단을 해제해야 합니다.
SECTION 07

수업 운영 추천 흐름

45분 1차시

  • 0~5분: 접속과 개인정보 주의 안내
  • 5~15분: STEP 1 학생부 기록 요약
  • 15~25분: STEP 2 흐름 분석
  • 25~35분: STEP 3 AI 프롬프트 생성
  • 35~45분: 후보 주제 1개 이상 메모

2차시 이상

  • 1차시: STEP 1~4 완료
  • 2차시: 위험도 점검과 별점 평가
  • 3차시: 계획서 완성 및 기록지 작성
  • 마무리: JSON/CSV 제출 확인
SECTION 08

자주 묻는 질문

Q. 자동 저장만 믿어도 되나요?

자동 저장은 브라우저 임시 저장입니다. 기기 변경, 캐시 삭제, 브라우저 초기화 상황에 대비해 JSON 저장을 병행하는 것이 안전합니다.

Q. 학생부 내용을 그대로 붙여넣어도 되나요?

그대로 붙여넣지 않습니다. 활동명, 배운 점, 아쉬운 점, 다음 질문만 요약합니다. 이름, 주민번호, 민감 정보는 입력하지 않습니다.

Q. AI가 만든 주제를 그대로 제출해도 되나요?

그대로 제출하지 않습니다. AI 결과는 후보 생성용입니다. 반드시 학생 자신의 경험, 학교 수업, 교과 개념, 수행 가능성을 반영해 수정해야 합니다.

Q. DB 주제는 그대로 써도 되나요?

DB 주제는 참고자료입니다. 학교, 지역, 학생 경험, 수업 내용에 맞게 구체화해야 합니다.

검수 메모이 문서는 기존 사용법 파일의 구조를 유지하지 않고, 프로그램 원본 화면 구조를 기준으로 다시 작성했습니다. 일부 이미지는 정적 렌더링 기반이므로 실제 브라우저의 스크롤 위치·애니메이션·포커스 상태와는 다를 수 있습니다.
